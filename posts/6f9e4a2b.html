从 PPO 到 GRPO：大模型强化学习的目标函数、训练流程与推理涌现

发表于 2026-07-12|更新于2026-07-12|大模型
|字数总计:4261|阅读时长:14分钟|阅读量:

强化学习进入语言模型训练，并不是给预训练再加一个神秘模块。它做的是一件更具体的事：当“下一个词预测得像训练文本”与“这个回答对人有帮助、正确且合乎约束”不是同一个目标时，怎样把后者变成可优化的信号，同时又不让策略一步走得太远。本文只依照四篇原论文梳理这条线：PPO 给出稳定更新的基本形状，InstructGPT 把人类比较接到这个形状上，DeepSeekMath 用组内相对回报减掉 Critic，DeepSeek-R1 则展示了可验证奖励在推理任务上的一种训练路径。

为什么语言模型需要强化学习

从下一个词到一条轨迹

自回归语言模型在给定提示词 \(x\) 后生成 \(y=(y_1,\ldots,y_T)\)。把它写成强化学习问题时，第 \(t\) 步状态、动作与策略可以记为

\[ s_t=(x,y_{<t}),\qquad a_t=y_t,\qquad \pi_\theta(a_t\mid s_t)=\pi_\theta(y_t\mid x,y_{<t}). \]

一段完整回答就是一条轨迹。奖励可以在结束时按整段回答给出，也可以拆成逐 token 的项。这个映射并不宣称文本生成天然就是某种真实环境中的决策过程；它只是让我们能够明确地问：哪些输出应被提高概率，更新时又该离旧策略多远。

预训练最大化的是语料上的似然，而用户偏好往往是比较性的、任务相关的，甚至要求满足格式或可验证答案。因此，强化学习阶段通常从一个已监督微调的策略出发，使用外部反馈定义回报。在 InstructGPT 的具体设置里，提示词到回答被作为 bandit 式的单回合环境：给定 prompt 和 response 后得到奖励并终止。这样的简化很适合解释 RLHF，也提醒我们不要把所有语言现象都硬套成长期环境交互。

优势不是“答案分数”

策略梯度不只需要一个回报，还需要知道某个动作相对基线是好还是坏。记估计优势为 \(\hat A_t\)：它为正，说明该动作比基线好；为负，则说明应降低它的概率。PPO、RLHF 和 GRPO 的差异，很大一部分就在这个基线从哪里来：可以来自价值函数，也可以来自同题多个回答的平均分。

PPO：限制策略更新幅度

概率比与截断目标

PPO 把新旧策略在同一动作上的变化写成重要性比率：

\[ r_t(\theta)=\frac{\pi_\theta(a_t\mid s_t)}{\pi_{\theta_{\mathrm{old}}}(a_t\mid s_t)}. \]

若直接最大化 \(r_t(\theta)\hat A_t\)，正优势动作可能被过度放大，负优势动作也可能在一次更新中被过度压低。PPO 的 clipped surrogate objective 是

\[ L^{\mathrm{CLIP}}(\theta)=\hat{\mathbb E}_t\left[ \min\left(r_t(\theta)\hat A_t,\; \operatorname{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat A_t\right) \right]. \]

这里 \(\theta\) 是当前策略参数，\(\theta_{\mathrm{old}}\) 是采样数据的旧策略，\(\epsilon\) 是允许的概率比变化窗口。取两个项的较小值，使目标在“策略正在因这个样本而得到更多收益”的方向上停止继续奖励越界的比率；原论文将它描述为未截断 surrogate 的悲观下界。它不是严格禁止概率变化，而是在最容易造成过大更新的方向上削平额外收益。

图 1：PPO 原论文第 3 页、Equation (7) 的 clipped surrogate objective。截图仅用于论文解读，来源：Schulman et al., 2017。

Critic、GAE 与 KL 各管什么

经典 PPO 是 actor-critic 方法：策略是 actor，价值函数 \(V_\psi(s_t)\) 估计从状态出发的回报，作为优势的基线；GAE 则利用时间差分残差构造较低方差的 \(\hat A_t\)。这三者分工不同：PPO 的截断限制策略比率，Critic 为优势估计提供基线，GAE 在偏差与方差之间调节估计。

PPO 原论文还讨论了 KL penalty，并按目标 KL 自适应调整惩罚系数来约束更新。到了语言模型微调，常见的 KL 参照物不再只是上一轮策略，而是一个冻结的参考模型。二者的共同直觉是相同的：奖励再高，也不能让分布无代价地漂移；但“截断比率”和“相对参考策略的 KL”是可并用、也应区分的两类约束。

RLHF：把人类比较变成奖励信号

SFT、Reward Model 与 PPO

InstructGPT 将流程分为三步：先用人工示范进行 supervised fine-tuning（SFT），再让标注者对同一 prompt 的多个回答排序以训练 reward model（RM），最后用 PPO 优化策略以获得较高 RM 分数。SFT 给出一个可用的初始语言策略；RM 将“回答 A 比回答 B 好”压缩为标量偏好；PPO 则在可采样的策略上实际执行优化。

图 2：InstructGPT 原论文第 3 页、Figure 2。蓝色箭头表示各阶段使用的数据；截图仅用于论文解读，来源：Ouyang et al., 2022。

成对偏好如何成为标量奖励

对 prompt \(x\)，令 \(y_w\) 是标注者偏好的回答、\(y_l\) 是较差回答，\(r_\phi(x,y)\) 是奖励模型输出。论文使用 Bradley-Terry 形式的负对数似然：

\[ \mathcal L_{\mathrm{RM}}(\phi)= -\mathbb E_{(x,y_w,y_l)\sim D} \left[\log \sigma\big(r_\phi(x,y_w)-r_\phi(x,y_l)\big)\right]. \]

其中 \(D\) 是人工比较数据，\(\sigma\) 是 sigmoid。奖励零点可整体平移；真正进入 Bradley-Terry 似然的是分数差，它对应偏好回答胜过非偏好回答的 log-odds。因此不能把它概括为只有排序重要：整体缩放会改变这些 log-odds 的强度。这也解释了为什么奖励模型的泛化和失配会成为 RLHF 的关键风险。

带参考策略的 RLHF 目标

InstructGPT 在 PPO 阶段把 RM 分数作为标量回报，并对每个 token 加入相对 SFT 模型的 KL 惩罚。写成常见的轨迹形式，可概括为

\[ J_{\mathrm{RLHF}}(\theta)= \mathbb E_{x,\,y\sim\pi_\theta} \left[r_\phi(x,y)-\beta\sum_{t=1}^{T} \log\frac{\pi_\theta(y_t\mid x,y_{<t})} {\pi_{\mathrm{SFT}}(y_t\mid x,y_{<t})}\right]. \]

\(\beta\) 控制对参考策略偏离的代价，\(\pi_{\mathrm{SFT}}\) 是冻结的初始监督策略。括号中的逐 token 对数比是采样路径上的 KL 型正则项：它让“骗过 RM 的高分”必须支付远离语言先验的代价。论文还报告 PPO-ptx：在这个目标之外混入预训练分布上的对数似然梯度，以缓解部分公共 NLP 任务的性能回退。这里应把它理解为论文中的具体训练设计，而不是 RLHF 必然自动保留所有原能力的保证。

GRPO：用组内相对奖励替代 Critic

同题采样一组回答

DeepSeekMath 的出发点很直接：LLM 的 PPO 中，价值函数通常与策略模型量级相近，训练它会带来显著显存和计算开销；而许多结果奖励只在回答末尾出现，也使逐 token 价值估计更难。GRPO 对同一问题 \(q\) 从旧策略采样 \(G\) 个输出 \(\{o_i\}_{i=1}^{G}\)，以组内平均回报作为基线，而不是训练额外 Value Model。

对第 \(i\) 个回答的奖励 \(r_i\)，outcome supervision 下的组相对优势为

\[ \hat A_i=\frac{r_i-\operatorname{mean}(r_1,\ldots,r_G)} {\operatorname{std}(r_1,\ldots,r_G)}. \]

同一回答中的 token 可共享这个终局优势。它表达的不是“这个答案绝对值几分”，而是“在同题这批样本里它高出平均多少个标准差”。随后目标仍保留 PPO 式的 ratio clipping，并加入当前策略相对参考策略的 KL 正则；区别在于 DeepSeekMath 把 KL 直接放进 loss，而不是先混进奖励再计算优势。

图 3：DeepSeekMath 原论文第 13 页、Figure 4。GRPO 去除了 value model，以 group computation 估计基线；截图仅用于论文解读，来源：Shao et al., 2024。

节省的是什么，交换的又是什么

GRPO 去掉的是单独训练的价值模型，不是不要约束的“裸策略梯度”：参考策略与 KL 正则仍在，PPO 式截断也仍在。它用同题多样本换取基线，适合奖励天然具有比较意义的情形；DeepSeekMath 明确指出，这与奖励模型常由同题比较数据训练的性质相契合。

代价也很清楚。每个问题要生成一个组，采样预算成为算法的一部分；而相对优势取决于组内分布。由上式可直接看出，若组内回报方差很低，分母很小，归一化相对估计会变得敏感，训练可能不稳定。这是从该估计式得到的数值分析结论，不是论文报告的本站实验结果。实际训练还必须面对奖励模型本身的偏差，或者在可验证任务中面对验证器覆盖范围的限制。

DeepSeek-R1：可验证奖励与推理行为

R1-Zero 的规则奖励

DeepSeek-R1 论文以 GRPO 训练 R1-Zero，并强调它在初始阶段只用强化学习而不先做监督微调。对于数学、代码和逻辑推理，论文的规则奖励由两部分组成：

\[ R_{\mathrm{rule}}=R_{\mathrm{acc}}+R_{\mathrm{format}}. \]

\(R_{\mathrm{acc}}\) 从指定格式提取最终答案后做规则正确性验证，例如对确定性数学答案比对标准答案，或检查代码是否通过预定义测试；独立的 \(R_{\mathrm{format}}\) 约束回答把推理过程放在 <think>...</think> 标签中。论文明确说明两项等权，并在这类推理任务上避免使用神经奖励模型，理由是其在大规模 RL 中可能被 reward hacking。

图 4：DeepSeek-R1 v2 原论文第 4 页、Figure 1。图中展示 R1-Zero 的 AIME 轨迹及平均回答长度；截图仅用于论文解读，来源：DeepSeek-AI et al., 2025。

图中能读出什么，不能读出什么

论文报告 R1-Zero 在 AIME 2024 上的 pass@1 平均分由 15.6% 升至 77.9%；同一图还显示，训练中平均回答长度增长，并在约 8.2k step 附近与性能一起出现跃迁。DeepSeek-R1 v2 在该点附近把最大生成长度从 32,768 提高到 65,536，性能与长度跃迁伴随这一配置变化，不能作为纯内生涌现的证据。论文将这种更长的生成称为更多 thinking time，并报告反思、替代方案探索等行为。

这些是论文训练轨迹上的观察和指标，不是“模型已经具有人的推理过程”的证明。更长的 trace 可以是为了得到可验证答案而学到的有效搜索或表达策略，也可能包含冗余、模板化或迎合格式奖励的部分。将可观察的文本行为等同于人类式推理，会越过这四篇论文实际提供的证据边界。

从冷启动到第二阶段 RL

R1-Zero 的可读性和语言混杂问题，促使论文给出 DeepSeek-R1 的多阶段管线：先收集数千条具备对话式、人类对齐思维过程的 cold-start 数据并进行 SFT；随后进行第一阶段 RL，改善推理表现与语言一致性；再以 rejection sampling 生成和筛选数据，并将推理与非推理数据一起做第二次 SFT；最后进行第二阶段 RL，在推理数据上沿用规则奖励、在通用数据上使用奖励模型，以兼顾推理、helpfulness 与 harmlessness。

图 5：DeepSeek-R1 v2 原论文第 6 页、Figure 2。R1 Dev1、Dev2、Dev3 是管线中的中间检查点；截图仅用于论文解读，来源：DeepSeek-AI et al., 2025。

这条管线的含义并非“RL 替代了监督学习”。恰恰相反，论文把可验证推理奖励、冷启动示例、拒绝采样和覆盖非推理场景的 SFT 组合起来，处理能力、可读性和通用偏好的不同目标。它也保留了一个工程上的现实：能可靠检查的任务和开放式人类偏好任务，需要的奖励来源并不相同。

统一视角与工程判断

一张表看四种训练位置

方法或阶段 数据来源 奖励 Critic 优势基线 参考约束 主要权衡
PPO 环境采样轨迹 环境回报 通常需要价值函数 \(V_\psi\) 与 GAE 截断比率；可配 KL 约束 更新稳定性与样本效率之间的折中
InstructGPT RLHF 人工示范、回答比较、prompt RM 标量分数 论文中 value function 由 RM 初始化 价值估计 相对 SFT 的逐 token KL；PPO-ptx 另混入预训练梯度 人类偏好可优化，但 RM 可能失配或被过度优化
DeepSeekMath GRPO 同题采样回答与奖励模型评分 outcome 或 process reward 不训练单独 value model 同题组均值与标准差 相对 reference policy 的 KL 直接入 loss 节省 Critic 资源，但需组采样且依赖组内区分度
DeepSeek-R1 推理题、冷启动数据、拒绝采样与通用数据 推理阶段规则奖励；通用阶段奖励模型 GRPO 不设独立 Critic 组相对奖励 GRPO 的 reference-policy regularization 可验证性强的域收益显著，开放域仍需偏好与安全信号

表中的“数据来源”说的是论文各阶段直接优化时使用的数据，而非模型全部预训练语料。表中的“优势基线”也不等于奖励本身：奖励回答“什么更好”，基线回答“相对什么算更好”。把两者混为一谈，容易误解 GRPO 为什么能去掉 Critic，也容易误解 KL 为什么没有被一起删掉。

设计目标比算法名字更重要

从四篇论文可以抽出一个实用的检查顺序。第一，奖励是否可信：人类比较能覆盖开放式质量，但有成本和偏好偏差；可验证奖励精确，却只覆盖答案能被规则或测试判定的子域。第二，优势是否稳定：价值函数、GAE 和组内标准化都在解决估计问题，只是付出的模型、采样和方差成本不同。第三，策略漂移是否可控：PPO 的 clip 与参考模型 KL 都是在避免短期高分牺牲已有语言能力和分布质量。

因此，不能只因一个算法去掉 Critic 就推断它必然更优，也不能只因图上答案更长就推断其内部机制已被解释。工程选择应从奖励可验证性、可承受采样量、参考策略约束和目标任务覆盖面出发，而不是从算法名称出发。

总结与论文索引

PPO 的核心贡献是把“别离旧策略太远”写进可批量优化的截断目标；InstructGPT 展示了 SFT、偏好奖励模型与带 KL 的 PPO 如何连接；GRPO 将基线改为同题组内相对回报，减去独立 Critic 的负担而保留参考策略约束；DeepSeek-R1 则把这套机制放到可验证推理奖励和多阶段训练中。它们共同说明，强化学习的能力来自奖励、估计和约束的配合，而不是一个孤立公式。

局限性与阅读边界

  • 更长的推理 trace、反思措辞或更高分数是论文中的行为观察，不构成人类式推理已经被证明的结论。
  • 可验证奖励在数学、代码和逻辑等可判定任务上尤其有力，但其适用范围受验证器和任务定义限制，不能直接覆盖开放式写作、价值判断或全部安全问题。
  • GRPO 的相对估计依赖组内回报方差；当方差很低时，标准化分母会使估计敏感，必须把这一数值风险与采样设计一起考虑。
  • 无论奖励来自 RM 还是规则评估器，优化都会寻找评估器的薄弱处。InstructGPT 的偏好模型失配与 DeepSeek-R1 对神经奖励模型 reward hacking 的讨论，都说明高奖励不自动等于目标行为被完整实现。
